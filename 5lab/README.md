# Лабораторная работа №5  
**Тема:** Циклический избыточный код. CRC

## Цель работы
1. Получить представление о том, как осуществляется проверка на наличие ошибок в пакетах с данными в современных системах связи (Error detection) посредством использования циклического избыточного кода CRC (Cyclic Redundancy Check).
2. Реализовать алгоритм вычисления CRC на языке C++.
3. Проверить корректность передачи данных с использованием CRC на приёмной стороне.
4. Оценить способность выбранного порождающего полинома обнаруживать одиночные битовые ошибки.

## Ход работы

### 1. Выбор исходных данных
- Номер варианта: 20
- Порождающий полином: $$G(x) = x^7 + x^6 + x^5 + x^3 + x^2 + x$$
  - двоичное представление: 11101110
- Длина исходного пакета:
  - N=20+20=40 бит
  - N=250 бит
  - Длина CRC: 7 бит (степень полинома = 7)

### 2. Реализация вычисления CRC (C++)
- Реализована функция `compute_CRC`, выполняющая:
  - Копирование исходных данных в буфер с добавлением 7 нулей.
  - Побитовое деление с использованием операции XOR.
  - Извлечение остатка длиной 7 бит как CRC.
- CRC добавлен к исходному пакету, сформирован полный пакет длиной *`N+7`*

### 3. Проверка на приёмной стороне
- Реализована функция `verify`, выполняющая:
  - Деление всего полученного пакета (данные + CRC) на тот же порождающий полином.
  - Анализ остатка: если все 7 бит = 0 → ошибок нет; иначе — ошибка обнаружена.
- Проведена проверка целостности корректного пакета — остаток = 0, ошибок не обнаружено.

### 4. Моделирование одиночных ошибок
- Реализована функция `verify_count`:
  - Выполняет цикл из 250+7=257 итераций.
  - На каждой итерации инвертирует один бит в полном пакете (от 0 до 256).
  - Проверяет каждую искажённую версию с помощью `verify`.
  - Подсчитывает количество обнаруженных и необнаруженных ошибок.

### 5. Анализ результатов
- Для пакета из 250 бит:
  - Обнаружено ошибок: 247
  - Не обнаружено ошибок: 10
- Причина пропуска ошибок: выбранный полином не содержит свободного члена, $$x^0$$ что снижает его способность обнаруживать опеределенные одиночные ошибки.
